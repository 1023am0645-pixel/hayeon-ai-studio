## 1. 답변을 "사라지는 토스트" → "채팅 기록"으로

현재: AI 답변을 showToast로 띄워서 몇 초 후 사라짐. 가독성 낮음.
목표: 직원 상세 패널의 "대화하기" 영역에 **스크롤되는 대화 로그**로 누적 표시(사용자 말풍선 + AI 말풍선).

구현 가이드:
- 상태에 대화 보관: `state.chat = state.chat || {}; state.chat[empId] = state.chat[empId] || []`
- chatForm 제출 시:
  1) `state.chat[empId].push({ role: "user", text: message })` → 렌더
  2) 임시 `{ role: "ai", text: "…", pending: true }` 추가 → 렌더 (타이핑 표시)
  3) `const text = await requestEmployeeReply(employee, message)`
  4) pending 항목을 `{ role:"ai", text }`로 교체 → 렌더, `saveState()`
  5) 실패 시 pending을 `연결 오류` 또는 createSimulatedReply로 교체
- 답변에는 showToast 사용하지 말 것(로그로만).
- 마크업 예:
```html
<div class="chat-log" id="chatLog">
  <!-- 메시지: <div class="chat-msg user|ai">텍스트</div> -->
</div>
```
- CSS 예(가독성 ↑, 다크 배경):
```css
.chat-log{max-height:260px;overflow-y:auto;display:flex;flex-direction:column;gap:8px;padding:10px;border-radius:12px;background:rgba(18,12,7,.92)}
.chat-msg{max-width:85%;padding:8px 11px;border-radius:12px;font-size:.92rem;line-height:1.5;white-space:pre-wrap;word-break:break-word}

.chat-msg.ai{align-self:flex-start;background:#2a1c10;color:#f6ecd6;border:1px solid rgba(255,232,184,.25)}
```

## 3. 오케스트레이션 — "지시 1번 → 관련 직원 자동 분배 → 각자 수행"

목표: 사용자가 한 줄 지시("다음달 신입 입문교육 준비") → 매니저가 관련 직원 선정 → 각 직원이 자기 역할로 수행 → 결과 취합 + 보드 등록.

### 3-1. 백엔드: worker.js에 매니저 라우트 추가(또는 /api/agent 재사용)
매니저 호출은 기존 /api/agent에 "매니저 system 프롬프트"를 넣어 그대로 사용 가능.

### 3-2. 프론트 오케스트레이터 함수 (ai-adapter.js 또는 app.js)
```js
// 1) 매니저: 목표 → 어떤 직원에게 무슨 세부과제를 줄지 JSON으로
async function planTasks(goal, employees) {
  const roster = employees.map(e => `- ${e.id} | ${e.name} | ${e.role}`).join("\n");
  const system =
    "너는 AI 스튜디오의 총괄 매니저다. 아래 직원 명단 중, 주어진 목표에 꼭 필요한 직원만 골라 " +
    "각자에게 줄 구체적 세부지시를 만든다. 반드시 JSON 배열만 출력: " +
    '[{"employeeId":"...","subtask":"..."}]. 불필요한 직원은 넣지 마라.\n\n[직원 명단]\n' + roster;
  const res = await fetch("/api/agent", {
    method:"POST", headers:{"content-type":"application/json"},
    body: JSON.stringify({ system, user: goal })
  });
  const { text } = await res.json();
  // JSON 파싱(앞뒤 잡텍스트 제거)
  const m = text.match(/\[[\s\S]*\]/);
  return m ? JSON.parse(m[0]) : [];
}

// 2) 분배 실행: 각 직원이 자기 역할로 수행
async function runOrchestration(goal) {
  const plan = await planTasks(goal, employees); // employees = 전체 직원 배열
  const results = [];
  for (const item of plan) {
    const emp = employees.find(e => e.id === item.employeeId);
    if (!emp) continue;
    setEmployeeStatus(emp.id, "working");           // 걷기/근무 표시(있다면)
    try {
      const text = await requestEmployeeReply(emp, item.subtask);
      results.push({ emp, subtask: item.subtask, text });
      // 보드에 등록하고 싶으면: createDirectedTask 등 기존 함수 활용
    } catch (e) {
      results.push({ emp, subtask: item.subtask, text: "(실패) " + e.message });
    }
  }
  return results; // 결과를 새 패널/모달에 직원별로 렌더
}
```
- UI: 상단이나 하비서 카드에 **"전 직원에게 지시"** 버튼 추가 → 입력창 → `runOrchestration(goal)` → 결과를 직원별 카드로 표시.
- 결과를 할 일판(task)에 자동 등록하려면 각 결과마다 기존 task 생성 함수 호출(상태 doing).

### 3-3. 모델 주의
- 무료 Workers AI(llama-3.1-8b-instruct-fp8)는 **엄격한 JSON 출력이 가끔 깨짐**. 위 정규식 파싱으로 1차 방어.
- 더 안정적이려면: (a) 매니저만 Gemini 무료 등급으로, 또는 (b) 키워드→부서 규칙 기반 라우팅을 폴백으로 둠.
  - 규칙 폴백 예: 목표에 "교안"→교안봇, "강의 설계/흐름"→연PD, "기획"→강의기획, "일정"→일정봇, "이벤트"→(해당 직원) 등 매핑.
