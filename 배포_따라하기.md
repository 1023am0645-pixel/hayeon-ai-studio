# 직원 AI 켜기 — 따라 하기 (개발 몰라도 OK · GitHub 방식)

드래그 업로드는 백엔드 함수(AI 연결)를 지원하지 않아요.
그래서 **GitHub(무료) → Cloudflare 연결** 방식으로 합니다. 전부 웹에서 클릭/드래그로 돼요.
명령어·코딩 없음, 결제수단 없음. **한 번만** 설정하면 끝.

올릴 폴더: `바탕화면 / 바이브코딩 / HAYEON AI STUDIO / _deploy`

---

## A. GitHub에 코드 올리기 (웹에서 드래그)

1. **github.com** 접속 → 무료 회원가입 / 로그인
2. 오른쪽 위 **+** → **New repository(새 저장소)**
3. Repository name: `hayeon-ai-studio` 입력 → 아래 **Create repository** 클릭
   - (Public/Private 아무거나 OK. 코드에 비밀번호·키 없어서 Public도 안전)
4. 새로 생긴 저장소 화면에서 가운데 **"uploading an existing file"** 글자 링크 클릭
   (안 보이면 **Add file → Upload files**)
5. 컴퓨터에서 `_deploy` 폴더를 열고, **그 안의 항목 전부**(index.html, src, assets, functions, wrangler.toml)를
   선택해서 업로드 칸으로 **드래그**
   - 폴더 구조 그대로 올라가요. 파일 좀 많아도(50여 개) 기다리면 됩니다.
6. 화면 아래로 스크롤 → 초록 **Commit changes** 클릭
   → 저장소에 파일들이 보이면 성공

## B. Cloudflare에 연결해서 배포

1. **dash.cloudflare.com** → 왼쪽 **Workers & Pages** → **Create** → **Pages** 탭
2. **Connect to Git** 선택 → GitHub 로그인/권한 허용(Authorize)
3. 방금 만든 저장소 `hayeon-ai-studio` 선택 → **Begin setup**
4. 빌드 설정은 이렇게 (중요):
   - **Framework preset**: `None`
   - **Build command**: (비워둠)
   - **Build output directory**: `/`  (슬래시 하나)
5. **Save and Deploy** 클릭 → 잠시 후 `https://....pages.dev` 주소 생성

## C. 무료 AI 켜기 (바인딩 확인)

대부분 `wrangler.toml` 덕분에 자동으로 켜져요. 확인만:
1. 프로젝트 → **Settings → Functions(또는 Bindings)**
2. **Workers AI** 바인딩에 이름 **`AI`** 가 있는지 확인
   - 없으면 **Add binding → Workers AI → 이름 `AI` → Save**
3. 추가했으면 **Deployments → 최근 배포 ⋯ → Retry deployment**

## D. 테스트 ✅

1. `https://....pages.dev` 열기
2. 직원 클릭 → 채팅창에 "이번 주 강의 우선순위 3개 정리해줘"
3. "생각 중…" 후 진짜 답이 오면 완료! 🎉

---

## 안 될 때
- 답이 "(오프라인)"이면 → C단계의 `AI` 바인딩 확인 + 재배포
- 빌드 실패하면 → B-4의 build output을 `/` 로 했는지 확인
- 막히면 화면 캡처해서 저(클로드)한테 가져오기

## 돈 / 보안
- Workers AI는 무료 한도 안에서만 동작, 넘으면 멈춤(자동 결제 없음)
- 코드에 API 키 없음 → 유출 위험 없음. 함수는 우리 사이트에서만 호출되게 막아둠
- GitHub에 올라가는 코드에도 비밀정보 없음

## 나중에 좋은 점
- 이렇게 GitHub로 연결해두면, 앞으로 수정사항이 생겼을 때 GitHub만 바꾸면 사이트가 **자동 재배포**돼요.
