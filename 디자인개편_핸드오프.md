## A. 테마 시스템 (디자인 토큰 + 전환 + 저장)

### A-1. 토큰 변수 (3테마)
루트(`html` 또는 `body`)에 `data-theme` 속성으로 전환. 컴포넌트는 변수만 사용.
```css
:root, [data-theme="aurora"]{           /* 기본 */
  --bg: linear-gradient(160deg,#d6e9fb,#eef0fb 50%,#fbe8f1);
  --surface: rgba(255,255,255,.66);
  --surface-2: rgba(255,255,255,.78);
  --border: rgba(255,255,255,.85);
  --text: #2b4a63; --text-dim: #7d93a3;
  --accent: #3f7ea8; --accent-2:#ff9d76;
  --shadow: 0 12px 30px rgba(120,150,180,.18);
  --blur: blur(10px);
}
[data-theme="light"]{
  --bg: linear-gradient(160deg,#eef2f8,#f7f9fc);
  --surface:#ffffff; --surface-2:#f6f9fd; --border:#e6eaf1;
  --text:#0f172a; --text-dim:#7a8699;
  --accent:#2563eb; --accent-2:#3f7ea8;
  --shadow:0 10px 26px rgba(20,40,80,.06); --blur:none;
}
[data-theme="dark"]{
  --bg:#0c1320;
  --surface:#141d2e; --surface-2:#0f1828; --border:#233048;
  --text:#eaf1fb; --text-dim:#8198b8;
  --accent:#4f7cff; --accent-2:#22d3ee;
  --shadow:0 12px 30px rgba(0,0,0,.4); --blur:none;
}
```

### A-2. 컴포넌트는 토큰만 사용 (예)
```css
.ws-panel{ background:var(--surface); border:1px solid var(--border);
  border-radius:16px; box-shadow:var(--shadow); backdrop-filter:var(--blur); color:var(--text); }
.ws-panel .title{ color:var(--text); }
.ws-muted{ color:var(--text-dim); }
.btn-primary{ background:var(--accent); color:#fff; }
.pill-doing{ background:color-mix(in srgb,var(--accent),transparent 86%); color:var(--accent); }
body{ background:var(--bg); }
```
→ 기존 베이지/갈색 하드코딩을 토큰으로 치환.

### A-3. 전환 UI + 저장
상단 버튼에 테마 토글(아이콘) 또는 작은 드롭다운:
```js
function setTheme(t){ document.documentElement.dataset.theme=t; localStorage.setItem("ui-theme",t); }
setTheme(localStorage.getItem("ui-theme") || "aurora");   // 부팅 시 기본=aurora
// 토글 버튼: aurora→light→dark 순환 or 드롭다운에서 선택
```

## D. 업무 보드 필터 + 항목 클릭 상세/문서 (#3)

- 상단 세그먼트 탭: **할 일 / 진행 중 / 완료** → `state.boardFilter`로 task 필터.
```js
const shown = state.tasks.filter(t => state.boardFilter==="all" || t.status===state.boardFilter);
```
  (todo→할 일, doing→진행 중, done→완료)
- 각 task 카드 클릭 → 상세 모달: 제목·담당자·상태·진행상황 + **저장된 산출물(결과 텍스트/문서) 열기**.
  - 오케스트레이션 결과를 task에 함께 저장해두면 됨: task 생성 시 `task.resultText` 또는 `task.docId`.
  - "문서 열기" → 결과 텍스트를 모달/뷰로, 또는 .md 다운로드.

## E. 순차 업무 처리 (#2) — 앞 산출물을 다음 직원에게 전달

지금: 직원들이 독립 병렬처럼 각자 처리.
원하는 것: **기획 → 교안 → 사례 …** 앞 단계 결과를 다음 직원이 이어받기.

### E-1. 매니저가 '순서'까지 정하게
planTasks 프롬프트에 추가:
> "직원 간 의존이 있으면 실행 '순서'대로 배열하라. 뒤 직원은 앞 직원의 산출물을 입력으로 받는다고 가정하고 subtask를 작성하라."
(배열 순서 = 실행 순서로 사용)

### E-2. 실행 시 누적 컨텍스트 전달
```js
async function runSequential(goal, employees, {onUpdate}={}){
  const plan = await planTasks(goal, employees);
  const results = [];
  let context = "";                       // 앞 단계 산출물 누적
  for (let i=0;i<plan.length;i++){
    const it = plan[i]; const emp = employees.find(e=>e.id===it.employeeId); if(!emp) continue;
    const userMsg = context
      ? `${it.subtask}\n\n[이전 단계 산출물(참고)]\n${context}`
      : it.subtask;
    onUpdate?.({phase:"start", key:`${it.employeeId}#${i}`, employee:emp, subtask:it.subtask});
    let text; try{ text=await requestEmployeeReply(emp, userMsg); }
    catch(e){ text="(실패) "+(e.message||e); }
    results.push({employee:emp, subtask:it.subtask, text});
    context += `\n## ${emp.name}: ${it.subtask}\n${text}\n`;   // 다음 직원에게 누적 전달
    onUpdate?.({phase:"done", key:`${it.employeeId}#${i}`, employee:emp, text});
  }
  return results;
}
```
- 검토 게이트(needsReview)와 결합: 검토 필요 단계는 승인 후 실행하고, **승인된 결과까지 context에 반영**한 뒤 다음 단계 진행.
- UI에 "1 → 2 → 3" 단계 흐름(타임라인)으로 보여주면 순차임이 직관적.
