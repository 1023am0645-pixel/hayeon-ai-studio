# 오케스트레이션 상세 설계 (코덱스 구현용)

"지시 1번 → 매니저가 관련 직원 선정 → 각 직원이 자기 역할로 수행 → 결과 취합 + 할 일판 등록"

> 함수/변수명은 현재 코드 기준으로 코덱스가 맞춰 조정. (예: `employees`, `state`, `requestEmployeeReply`,
> `setEmployeeStatus`, `createDirectedTask`, `getEmployee`, `saveState`, `render` 등)

---

## 0. 전체 흐름 (4단계)

```
[사용자] "다음달 신입 입문교육 준비해줘"
   │
   ▼  (1) PLAN — 매니저 LLM
[하비서/관제봇] 목표 분석 → 필요한 직원 + 각자 세부지시(JSON)
   │
   ▼  (2) DISPATCH — 각 직원 LLM 병렬/순차 호출
연PD: 90분 흐름안 / 교안봇: 교안 초안 / 일정봇: 일정표 / 사례봇: 실습 ...
   │
   ▼  (3) AGGREGATE — 결과 취합
[관제봇] 직원별 산출물 요약 + 전체 체크리스트
   │
   ▼  (4) BOARD — 할 일판 자동 등록 (각 직원 status=working/doing)
```

## 2. (1) PLAN — 매니저 호출

매니저는 기존 `/api/agent`를 그대로 사용(별도 엔드포인트 불필요). system 프롬프트만 다르게.

```js
async function planTasks(goal, employees) {
  const roster = employees.map(e => `- ${e.id} | ${e.name} | ${e.role}`).join("\n");
  const system = [
    "너는 'HA:YEON AI STUDIO'의 총괄 매니저다.",
    "주어진 목표를 달성하기 위해, 아래 직원 명단에서 '꼭 필요한 직원만' 선정하고",
    "각 직원에게 그 사람의 역할에 맞는 '구체적이고 실행 가능한 세부지시'를 만든다.",
    "규칙:",
    "1) 반드시 JSON 배열만 출력. 설명/머리말 금지.",
    "2) 형식: [{\"employeeId\":\"<id>\",\"subtask\":\"<한국어 세부지시>\"}]",
    "3) employeeId는 반드시 명단의 id와 정확히 일치.",
    "4) 목표와 무관한 직원은 절대 넣지 마라. 보통 3~6명.",
    "",
    "[직원 명단]",
    roster,
    "",
    "[예시] 목표: \"다음달 신입 입문교육 준비\"",
    '[{"employeeId":"lecture-pd","subtask":"신입 대상 90분 입문교육 강의 흐름안(도입·본론·실습·마무리) 작성"},',
    ' {"employeeId":"prompt-engineer","subtask":"입문교육 핵심 개념 교안 초안 작성"},',
    ' {"employeeId":"ppt-designer","subtask":"교안 기반 슬라이드 구성/제목 문구 정리"},',
    ' {"employeeId":"case-developer","subtask":"신입이 따라할 실습 예시 3개 설계"},',
    ' {"employeeId":"schedule-bot","subtask":"교육 일정·준비 마감일 정리"}]'
  ].join("\n");

  const res = await fetch("/api/agent", {
    method:"POST", headers:{"content-type":"application/json"},
    body: JSON.stringify({ system, user: goal })
  });
  const data = await res.json();
  return parsePlan(data.text, goal, employees);
}

function parsePlan(text, goal, employees) {
  // 1차: 본문에서 JSON 배열 추출
  try {
    const m = (text||"").match(/\[[\s\S]*\]/);
    if (m) {
      const arr = JSON.parse(m[0]);
      const valid = arr.filter(x => employees.some(e => e.id === x.employeeId) && x.subtask);
      if (valid.length) return valid;
    }
  } catch (_) {}
  // 2차: 모델 실패 시 규칙 기반 폴백
  return ruleBasedPlan(goal, employees);
}
```

### 규칙 기반 폴백 (모델 JSON이 깨질 때 안전망)
```js
function ruleBasedPlan(goal, employees) {
  const g = goal.toLowerCase();
  const rules = [
    [["강의","교육","수업","특강","입문"], "lecture-pd", "강의 흐름/설계안 작성"],
    [["교안","자료","원고"],               "prompt-engineer", "교안·교육자료 초안 작성"],
    [["ppt","슬라이드","발표자료"],         "ppt-designer", "슬라이드 구성/문구 정리"],
    [["실습","사례","예시"],               "case-developer", "실습 사례/예시 설계"],
    [["오프닝","도입","첫인사"],           "opening-writer", "오프닝·도입 멘트 작성"],
    [["리허설","시간","발표연습"],         "rehearsal-coach", "리허설·시간배분 점검"],
    [["현장","준비물","장비","체크"],       "field-manager", "현장 준비 체크리스트"],
    [["일정","마감","스케줄"],             "schedule-bot", "일정·마감일 정리"],
    [["ax","서포터즈","과제"],             "ax-pm", "AX 과제 정리/관리"],
    [["보고","보고서"],                   "report-writer", "보고서 초안 작성"],
    [["후기","피드백","만족도"],           "feedback-analyst", "후기/피드백 분석"],
    [["기록","아카이브"],                 "archive-curator", "기록 정리"],
    [["앱","화면","ux"],                  "app-planner", "앱 기획/화면 정리"],
    [["자동화","반복","템플릿"],           "automation-bot", "자동화 흐름 설계"],
  ];
  const picked = [];
  for (const [kws, id, sub] of rules) {
    if (kws.some(k => g.includes(k)) && employees.some(e=>e.id===id) && !picked.find(p=>p.employeeId===id))
      picked.push({ employeeId:id, subtask:`${goal} — ${sub}` });
  }
  // 아무것도 안 걸리면 하비서가 단독 처리
  if (!picked.length) picked.push({ employeeId:"chief-assistant", subtask:goal });
  return picked;
}
```

## 3. (2)(4) DISPATCH + BOARD — 실행

```js
async function runOrchestration(goal, { onUpdate } = {}) {
  const plan = await planTasks(goal, employees);
  const results = [];
  for (const item of plan) {                 // 순차(무료 모델 rate limit 보호)
    const emp = getEmployee(item.employeeId);
    if (!emp) continue;
    setEmployeeStatus(emp.id, "working");     // 근무 표시(있다면)
    onUpdate && onUpdate({ phase:"start", emp, subtask:item.subtask });
    let text;
    try { text = await requestEmployeeReply(emp, item.subtask); }
    catch (e) { text = "(처리 실패) " + (e.message||e); }
    // 할 일판 등록 — 기존 task 생성 로직 재사용 (없으면 state.tasks.push)
    if (typeof createDirectedTask === "function") {
      try { createDirectedTask(emp.id, makeTaskForm(item.subtask)); } catch(_) {}
    } else {
      state.tasks.push({ id:"t"+Date.now()+Math.random().toString(36).slice(2,6),
        title:item.subtask, assigneeId:emp.id, status:"doing", priority:"medium",
        tags:["#오케스트레이션"], createdAt:new Date().toISOString() });
    }
    results.push({ emp, subtask:item.subtask, text });
    onUpdate && onUpdate({ phase:"done", emp, text });
  }
  saveState(); render();
  return results;
}

// createDirectedTask가 FormData를 받는 형태면 맞춰서:
function makeTaskForm(title){
  const fd = new FormData();
  fd.set("title", title); fd.set("priority","medium"); fd.set("dueDate",""); fd.set("tags","#오케스트레이션");
  return fd;
}
```

## 4. UI

- 진입점: 상단 툴바 또는 하비서(전략비서) 카드에 **"전 직원에게 지시"** 버튼.
- 클릭 → 입력 모달(목표 한 줄) → "분배 중…" 진행 표시 → 결과 패널.
- 결과 패널: 직원별 카드(아바타 + 이름 + 세부지시 + 답변), 맨 위에 관제봇 종합요약.
- 진행 표시는 `onUpdate` 콜백으로 "연PD 작성 중… → 완료" 식으로 갱신.
- 실행된 직원은 건물 뷰에서 status=working으로 바뀌어 보이면 더 살아있는 느낌.

마크업 예:

## 5. 모델/비용 전략
- 워커(각 직원) = 무료 Workers AI(`llama-3.1-8b-instruct-fp8`)로 충분.
- **매니저(planTasks)는 JSON 안정성이 중요** → 가능하면 Gemini 무료 등급 권장. 안 되면 위 규칙 폴백이 보완.
- 호출은 **순차**로(병렬 X) — 무료 등급 분당 제한 보호. 직원 3~6명이면 십수 초.
- 한 번 실행당 LLM 호출 = (매니저 1 + 직원 N + 요약 1). N=4면 6회.

## 6. 구현/테스트 순서
1. `planTasks`만 만들고 콘솔에 출력 확인(목표 입력 → JSON 잘 나오나).
2. JSON 깨질 때 `ruleBasedPlan` 폴백 동작 확인.
3. `runOrchestration` 연결 → 직원별 결과 콘솔 확인.
4. 할 일판 등록 연결.
5. UI(버튼·모달·결과패널) 연결.
6. (선택) 관제봇 요약 추가.
→ 각 단계 push 후 라이브 확인. 막히면 해당 코드/에러를 클로드에게.
